Tetris

by: Charlie Fredberg

This game was made for my level 2 project at The League of Amazing Programmers.
Click here to download and play!

1. Why I Made This Game

I don't have a major reason I chose to make Tetris. I just had the idea to make it, thought it would be fun, and did it.

2. What I Learned

When user testing, my instructor, Tyler, had the idea of making a few words in capital letters for my instruction state. I took his advice and it looks great!

3. Architecture of My Game

My game has a few different parts that interact. The first one is the different states, like in "League Invaders". The second one is the blocks. These are each their own class that extends a parent block class. The parent class allows all the others to have certain variables that are referenced in my game panel class.

4. Technical Implementation

For my row removal part, I had to access my 2d array to check if a row had been filled. I checked my row and column variables with a nested for loop. There are 10 blocks that need to be filled for a row to be filled, so I had my code check each row and if a block was filled then it would add 1 to a variable. Once it got to the end of a row it would see what the value of the variable is. If it is not 10 then the variable would be set back to 0. If it is 10 then the row would be cleared out, the current row would be stored in a variable, and a nested for loop would shift down all the necessary blocks.

5. Most Challenging Part

Making rotations in my game was VERY tedious but they weren't really hard. I would say the most challenging part of my game was the collisions. They took a long time to plan out and they had a lot of debugging.

6. Future Fixes and Updates

If I was to keep working on my game I would fix all of the bugs. There are a few bugs in my game but it is in a playable state. One of the bugs is that when rotating the blocks in certain places, they throw an error. To fix this, I just wouldn't allow rotations in certain column for the blocks. Each block would have its own limitations. This would require a lot of if statements.
